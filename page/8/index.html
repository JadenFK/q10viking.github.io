Compute tree height

发表于 2018-08-04 | 分类于 数据结构与算法

Task

You are given a description of rooted tree.Your task is to compute and output its height.Recall that the height of a tree is the maximum depth of a node,or the maximum distance from a leaf to the root.You are given an arbitrary tree,not necessarily a binary tree.

Constraints

1 $\le$n$\le$ $10^{5}$

Output

Output the height of the tree

Example

Sample-1:
Input

5
4 -1 4 1 1

Output:

3

Explannation:
The input means that there are 5 nodes with numbers from 0 to 4,node 0 is a child of node 4,node 1 is the root,node 2 is a child of node 4,node 3 and node 4 is a child of node 1.To see this,let us write numbers of nodes from 0 to 4 in one line and the numbers given in the input in the second line underneath:

0 1 2 3 4
4 -1 4 1 1

Now we can see that node 1 is the root,because -1 corresponds to it in the second line.Also,we know that the nodes number 3 and number 4 are children of the root node.Also we know nodes number 0 and number 2 are children of the node 4

Sample-2:
Input

5
-1 0 4 0 3

Output:

4

Alogrithm

1) 创建Node类，其中包括parent记录父节点，childs记录孩子节点（选择Linkedlist）,方法addChild添加孩子节点
2) 将输入的数据转换成Node对象，并存储在Nodes数组中
3) 构建树：遍历Nodes数组读取Node的parent值，如果为-1，则是root根节点,记录root在数组中的位置，否则将当前的Node加入到父节点的孩子中Nodes[parent].addChild(Node)
4) 计算height:使用BFS遍历树

a) 声明一个Queue队列，和两个计数器currentLevel,nextLevel分别记录当前层要处理的节点数和下一层要处理的节点数
b) root根节点入队列，初始化currentLevel=1,nextLevel=0，height=0;
c) 检测队列是否为空，为空则结束
d) 队列的第一个元素出队列，currentLevel减1表示当前层已经处理一个元素
e) 当出队列的元素的孩子加入到队列中，每加入一个孩子nextLevel加1，统计下一层要处理的元素个数
f) 检测currentLevel是否为0（当前层的元素是否都已经处理完毕），如果为0，则currentLevel=nextLevel,nextLevel=0,表示换一层处理，并且height+1;
g)返回c)步继续处理。

Alogrithm-2

  1. 一个树的节点也可以用键值对来表示，其中键表示当前节点，值是一个集合，存储着孩子，通过构建字典来建立一棵树
  2. 建树，通过输入的值，如果当前值是-1，则标记当前节点序列index是root根节点，否则dictionary[parent].add(index)将该节点加入到其父节点中
  3. 遍历：BSF,不过使用的数据结构是双端队列
    a）初始化双端队列 q=deque([root]),target=root（target表示每一层最后遍历的目标），active=0（等于1时表时遍历到了目标）,height=0计数
    b)检查队列是否为空，(不为空弹出左边元素，并查看当前元素是不是target,如果是激活active=1,)否则结束
    c)将该元素的所有孩子加入到队列的右边
    d）检查active是不是1，是更新target取当前队列的最右边的元素
    e)回到b

Check for balanced parentheses in an expression

Algorithm:
1) Declare a character stack S.
2) Now traverse the expression string exp.
a) If the current character is a starting bracket (‘(‘ or ‘{‘ or ‘[‘) then push it to stack.
b) If the current character is a closing bracket (‘)’ or ‘}’ or ‘]’) then pop from stack and if the popped character is the matching starting bracket then fine else parenthesis are not balanced.
3) After complete traversal, if there is some starting bracket left in stack then “not balanced”

前缀表达式转换中缀表达式

介绍

Consider the below expression: a op1 b op2 c op3 d
If op1 = +, op2 = , op3 = +
The compiler first scans the expression to evaluate the expression b c, then again scan the expression to add a to it. The result is then added to d after another scan.
The repeated scanning makes it very in-efficient. It is better to convert the expression to postfix(or prefix) form before evaluation.
The corresponding expression in postfix form is: abc*+d+. The postfix expressions can be evaluated easily using a stack. We will cover postfix expression evaluation in a separate post.

Algorithm

  1. Scan the infix expression from left to right.
  2. If the scanned character is an operand, output it.
  3. Else,
    …..3.1 If the precedence of the scanned operator is greater than the precedence of the operator in the stack(or the stack is empty), push it.
    …..3.2 Else, Pop the operator from the stack until the precedence of the scanned operator is less-equal to the precedence of the operator residing on the top of the stack. Push the scanned operator to the stack.
  4. If the scanned character is an ‘(‘, push it to the stack.
  5. If the scanned character is an ‘)’, pop and output from the stack until an ‘(‘ is encountered.
  6. Repeat steps 2-6 until infix expression is scanned.
  7. Pop and output from the stack until it is not empty.

在转换过程中，stack主要是用来处理符号，而不是存储操作数

使用ServerSocket

服务程序的基本生命周期：

  1. 使用一个ServerSocket()构造函数在一个特定端口创建一个新的ServerSocket
  2. ServerSocket使用其accept()方法监听这个端口的入站连接。accept()会一直阻塞，直到一个客户端尝试建立连接，此时accept将返回一个连接客户端和服务器的Socket对象
  3. 根据服务类型，会调用Socket的getInputStream()方法或getOutputStream()方法，或者这两个方法都调用，以获得与客户端通信的输入和输出流
  4. 服务器和客户端根据已协商的协议交互，直到要关闭连接。
  5. 服务器或客户端（或二者)关闭关闭连接
  6. 服务器返回步骤2，等待下一次连接
