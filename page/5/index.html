Hashing Table

With direct addressing,an element with key k is stored in slots k.With hashing,this element is stored in slot $h(k)$. A hash table requires much less storage than a direct address table

We use a hash function $h$ maps the universe $U$ of keys into the slots of a hash table $T[0,…,m-1]$

$$h:U-> { 0,1,…,m-1 }$$

The hash function reduces the range of array indices and hence the size of the array.Instead of a size of $|U|$,the array can have size $m$

Heap-2 Change Array into MAX-Heap

发表于 2018-08-13 | 分类于 数据结构与算法

算法导论6.3
We can use the procedure SiftDown in a bottom-up manner to convert an array A[1..n],where n=A.length,into a max heap.

Note that the elements in the subarray A[$(\lfloor n/2 \rfloor +1) .. n$] are all leaves of the tree,and so each is a 1-element heap to begin with.

The procedure BUID-MAX-HEAP goes through the remaining nodes of the tree and runs SiftDown on each one.

Pseudocode

Running time

n-element heap has height $\lfloor lg^n \rfloor$ and at most $\lceil n/2^{h+1}\rceil$,see the below picture:

The time required by SiftDown when called on a node of height h is O(h),so we can express the total cost of BUILD-MAX-HEAP as being bounded from above by:

$$\sum_{h=0}^{\lfloor lg^n \rfloor} \lceil \frac{n}{2^{h+1}} \rceil O(h) = O(n \sum_{h=0}^{\lfloor lg^n \rfloor} \frac{h}{2^h})$$

Because:

$$\sum_{i=0}^{\infty}kx^k= \frac{x}{(1-x)^2}$$

We evaluate the last summation by substituting $x=1/2$,yielding
$$\sum_{i=0}^{\infty} \frac{h}{2^h}= \frac{1/2}{(1-1/2)^2}=2$$

Thus,we can bound the running time of BUILD-MAX-HEAP as:

$$O(n\sum_{h=0}^{\lfloor lg^n \rfloor} \frac{h}{2^h})=O(n\sum_{h=0}^{\infty} \frac{h}{2^h})=O(n)$$
