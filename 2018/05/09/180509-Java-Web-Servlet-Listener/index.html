HttpSessionBindingListener

Causes an object to be notified when it is bound to or unbound from a session. The object is notified by an HttpSessionBindingEvent object.This may be as a result of a servlet programmer explicitly unbinding an attribute from a session, due to a session being invalidated, or due to a session timing out.

  1. When ever a particular type of object adding/removing/replaceing in session scode,if we want to perform certain activity then we should go for HttpSessionBindingListener.
  2. HttpSessionBindingListener is not require to configure in web.xml(也就是不需要< listener >标签).When ever we are adding an object in the session scope web container will check wheather the corresponding class implement HttpSessionBingdingListener or not If it implements then the conrresponding valueBound() will be executed;the same thing is applicable for removal and replacement also;
  3. When ever we are configuring both attributeListener and bindingListener then first BindingListener will be executed followed by AttributeListener.
