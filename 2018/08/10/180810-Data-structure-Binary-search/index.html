Binary search

发表于 2018-08-10 | 分类于 数据结构与算法

算法导论2.3-5
Given a sorted array arr[] of n elements,write a function to search a given element x in arr[].

A simple approach is to do liner search.The time complexity of above algorithm is O(n).Another approach to perform the same task is Binary search.

Binary Search

Search a sorted array by repeatedly dividing the search interval in half.Begin with an interval covering the whole array.If the value of the search key is less than the item in the middle of the interval,narrow the interval to the lower half.Otherwise narrow it to the upper half.Repeatedlly check until the value is found or the interval is empty.

Time complexity

The worst-case running time of binary search is $O(lg^n)$

Demonstrate

For the worst case,let us say we want to search for the number 13.

A sorted array of 16 elements

Selecting the middle element as pivot(length/2)

Since 13 is less than pivot,we remove the other half of the array.

Repeating the process for finding the middle element for every sub-array


So,for reaching one element from a set of 16 elements,we had to divide the array 4 times.

We can say that
$$16\times (\frac{1}{2})^4=1$$
Similarly,for n elements,
$$n \times (\frac{1}{2})^k=1$$
final result
$$n=2^k$$
Make our equation into:
$$log_2^n=k$$
