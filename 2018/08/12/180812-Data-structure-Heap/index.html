发表于 2018-08-12 | 分类于 数据结构与算法

Heap

The (binary) heap data structure is an array object that we can view as nearly complete binary tree .

Property:

  • The tree is completely filled on all levels except possibly the lowest,which is filled from left up to point.

  • Height: we define height to be equal to the number of edges on the longest path from the root to a leaf.In particular,the height of a tree that consists of one node is equal to 0 and the height of tree shown above is equal to 3.

  • A complete binary tree with n nodes has at most $O(log^n)$

  • When the array representation for storing an n-element heap,the leaves are the nodes indexed by $\lfloor n/2\rfloor+1,\lfloor n/2\rfloor+2,…,n$

  • Given the index i of a node,we can easily compute the indices of its parent,left child and right child.

    PARENT(i) return $\lfloor i/2 \rfloor$
    LEFT(i) return $2i$
    RIGHT(i) return $2i+1$

The indices of the complete binary tree allows us to store the whole complete binary tree in an array.This is advantage for us.

But What do we pay for these advantage?

$\blacksquare$ We need to keep the tree complete.

Which binary heap operations modify the shape of the tree?

$\blacksquare$ Only Insert and ExtractMax(Remove changes the shape by calling ExtractMax)

Binary max-heap

Binary max-heap: is a binary tree(each node has zero,one or two children) where the value of each node is at least the values of its children.—–In other words for each edge of the tree its top value is greater or equals than its bottom value.

$\blacksquare$ size is the size of the heap

$\blacksquare$ lmaxSize is the maximum number of elements in the heap

$\blacksquare$ H[1…maxSize] is an array of length maxSize where the heap occupies the first size elements.

Note: we store just the array H and also variable size and maxSize.So the tree is given to us implicitly.

Basic operations:

$\blacksquare$ GetMax

return the root value.

$\blacksquare$ SiftUp

We swap the problematic node with its parent util the property is satisfied (The property: on the edge,its top value is greater or equals to its bottom value)

$\blacksquare$ SiftDown

We swap the problematic node with larger child until the heap property is satisfied. Running time: O(tree height)

$\blacksquare$ ChangePriority

Change the priority and let the changed element sift up or sift down depends on whether its priority increased or decreased.

$\blacksquare$ Rmove

change the priority of the element to $\infty$,let it sift up and then extract maximum.

Keeping the Tree Complete(核心)

$\blacksquare$ Insert

To insert an element,insert it as leaf in the leftmost vacant position in the last level and let it sift up.


$\blacksquare$ ExtractMax

To extract the maximum value,replace the root by the last leaf and let it sift down.

Summary

The resulting implementation is:

$\blacksquare$ fast: all operations work in time $O(log^n)$ (GetMax even works in O(1))
$\blacksquare$ space efficient: we store an array of priorities;parent-child connections are not stored,but are computed on the fly.

$\blacksquare$ easy to implement: all operations are implemented in just a few lines of code.
