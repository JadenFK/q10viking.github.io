Creating a controller class

Controllers are the major players in Spring’s MVC framework.Their primary job is to

$\blacksquare$ handle HTTP requests
$\blacksquare$ either directly hand the request off to a view to render HTML(browser displayed)
$\blacksquare$ or to write data directly to the body of a reponse(RESTful)

@Controller annotaion serves to identify this class as a controller and to mark it as candidate for component-scanning so that Spring will discover it and automatically create an instance of the class as a bean in the Spring application context.

@RequestMapping annotaion,when applied at the class level,specifies the kind of requests that this controller handles.

Handing a GET request

The class-level $@RequestMapping$ specification is refined with(细化) the $@GetMapping$ annotaion that adorns the showDesignForm() method.Method-level $@GetMapping$,paired with the class level $@RequestMapping$,specifies that when an HTTP GET request is received for “/design”,the showDesignForm will be called to handle the request.

$@GetMapping$ is a relatively new annotaion,having been introduced in Spring4.3.Prior to Spring 4.3,you might have used a method-level $@RequestMapping$ annotaion instead：

Note:
Generally,I prefer to only use $@RequestMapping$ at the class level to specify the base path and to use the more spefic $@GetMapping$,$PostMapping$,etc. on each of the handler methods.

Model is an object that ferries data between a controller and whatever view is charged with rendering that data.Ultimately,data that is placed in Model attribute ends up being copied into the servlet response’s attribute for the view to find them.
