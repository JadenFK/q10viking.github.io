Java-8(1) Passing code with behavior parameterization

发表于 2018-09-01 | 分类于 Java基础

A well-known problem in software engineering is that no matter what you do,user requirements will change.

Behavior parameterization is a software development pattern that lets you handle frequent requirement changes.

  • it means taking a block of code and making it available without executing it.
  • This block of code can be called later by other parts of your program.
  • Which means that you can defer the execution of that block of code.

We start the tutorial by walking you through an example of how you can evolve your code to be more flexiable for changing requirements.

what if you want to alter the filter traversing to enhance performance?You now have to modify the implementation of all of your methods instead of a single one.This is expensive from an engineering effort perspective.

Behavior parameterization

Model your selection criteria: you’re working with apples and returning a boolean based on some attribute of Apple(for example,is it green? is it heavier than 150 g?).We call this a predicate(谓词)(that is,a function that returns a boolean)

Let’s therefore define an interface to model the selection criteria

You can now declare multiple implementations of ApplePredicate to represent different selection criteria

$\blacksquare$ What you did is related to the strategy design pattern,which lets you define a family of algorithms encapsulate each algorithm(called a strategy),and select an algorithm at run-time.
In this case the family of algorithms is ApplePredicate and different stategies are AppleHeavyWeightPredicate and AppleGreenColorPredicate

$\blacksquare$ how you can make use of the different implementations of ApplePredicate? You need your filterApples method to accept ApplePredicate objects to test a condition on an Apple. This is what behavior parameterization(行为参数化) means: the ability to tell a method to take mutilple behaviors( or strategies) as parameters and use them internally to accomplish different behaviors

$\blacksquare$ This has a great software engineering benefit,you can separate the logic of iterating the collection inside the filterApples method with the behavior you want to apply to each element of the collection.

filtering by abstract criteria

You can now create different ApplePredicate objects and pass them to the filterApples method.—-Free flexibility.
Such as you find all green apples that are heavier that 150 g.All you need to do is create a class that implements the ApplePredicate.

The behavior of the filterApples method depends on the code you pass to it via the ApplePredicate object.In other words,you’ve parameterized the behavior of the filterApples method.

Using Anonymous classes

At the moment,when you want to pass new behavior to your filterApples method,you’re forced to declare classes that implement the ApplePredicate interface and then instantiate ApplePredicate object that you allocate only once.

By using Anonymous class,that allow you to declare and imstantiate a class at the same time.—-concise

using Lambda expression

In Java 8,we can using lambda expression: it combine the flexiable of interface( strategy ) and concise of anonymous.
