Object Wrappers and Autoboxing

发表于 2018-09-03 | 分类于 Java基础

Most of the time,while working with numbers in java,we use primitive data types.But,java also provide various numeric wrapper(包装器) subclass under the abstract class Number present in java.lang package.

Properties of wrapper

  • The wrapper classes are immutable–you cannot change a wrapped value after the wrapper has been constructed.

  • They are also final ,so you cannot subclass them

Why use number wrappers

  • Constants defined by the number wrapper,such as $MIN_VALUE$ and $MAX_VALUE$,that provide the upper and lower bounds of the data type are very much useful.
  • The designers of Java found the wrappers a convenient place to put certain basic methods.(such as those for converting strings of digits to numbers)

Autoboxing and Auto-unboxing

Equality

As you know,the $==$ operator,applied to wrapper objects,only tests whether the objects have identical memory locations. The following comparision would probably fail.

However,a Java implementation may,if it choose,wrap commonly occuring values into identical objects,and thus the comparison might be true.

The remedy is call the equals method when comparing wrapper objects.

numeric parameter problem

Some people think that the wrapper classes can be used to implement methods that can modify numric parameters.However that is not correct.

Because Integer object is immutable.

If you want to write a method to change numeric parameters,you can use one of the holder types defined in org.omg.CORBA package: IntHolder,BooleanHolder, and so on.Each holder type has a public field value through which you can access the stored value.
