Method parameters

发表于 2018-09-03 | 分类于 Java基础

There are two terms of computer science that describe how parameters can be passed to a method in a programming language.

  • Call by value(按值传递) means that the method gets just the value that the caller provides.
  • Call by reference(按引用传递) means that the method gets the location of the variable that the caller provide.

The Java programming language always uses call by value

There are two kinds of method parameters

$\blacksquare$ Primitive types(numbers,boolean values)

$\blacksquare$ Object references

Parameters-primitive types

  1. $x$ is initialized with copy of the value of $percent$(that is,10)
  2. $x$ is tripled–it is now 30.But $percent$ is still 10
  3. The method ends,and the parameter variable $x$ is no longer in use.

Parameters-Object reference

Java programming language does not use call by refernce for objects. Instead,object references are passed by value

Exmaple-1

Implement a method that triples the salary of an employee.

  1. $x$ is initialized with a copy of the value of $harry$,that is,an object reference.
  2. The $raiseSalary$ method is applied to that object reference.The $Employee$ object to which both $x$ and $harry$ refer gets its salary raised by 200 percent.
  3. The method ends,and parameter variable $x$ is no longer in use.Of course,the object variable $harrt$ continues to refer to the object whose salary was tripled.

Example-2

Write a method that swaps two employee object

However,the method dose not actually change the object references that are stored in the variables $a$ and $b$.The $x$ and $y$ parameters of the $swap$
method are initialized with copies of these references.The method then proceeds to swap these copies.

But ultimately,this is wasted effort.When the method ends,the parameter variables $x$ and $y$ are abandoned.The original $a$ and $b$ still refer to the same objects as they did before the method call.

Sample Test

Summary

$\blacksquare$ A method cannot modify a parameter of a primitive type.

$\blacksquare$ A method can change the state of an object parameter

$\blacksquare$ A method cannot make an object parameter refer to a new object.
