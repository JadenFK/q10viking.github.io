java-8(2) Lambda expression

发表于 2018-09-04 | 分类于 Java基础

In the previous post : Java-8(1) Passing code with behavior parameterization .It lets you define a block of code that represents a behavior and then pass it around.

In this post,we teach you about a new feature in Java 8 that tackles anonymous class’s problem: verbose. Lamdba expression which let you represent a behavior or pass code in a concise ,flexiable way.

Lambda expression

A lambda expression can be understood as a concise representation of an anonymous function that can be passed around: it doesn’t have a name,but it has a list of parameters,a body,a return type,and also possibly a list of exceptions that can be thrown.

  • $Anonymous$— We say anonymous because it dosen’t have an exlicit name like a method would normally have.
  • $Function$— We says function because a lambda isn’t associated with a particular class like a method is.But like a method,a lambda has list of parameters,a body,a return type,and a possible list of exceptions that can be thrown.
  • $Passed \ around$— a lambda expression can be passed as argument to a method or stored in a variable
  • $Concise$—You don’t need to write a lot of boilerplate like you do for anonymous classes.

Lambda syntax

The basic syntax of lambda are:

  • expression

  • statements

Example

$\blacksquare$ $() \to { }$ This lambda has no parameters and return void.It’s similar to a method with an empty body: $public \ void \ run(){ \ }$

$\blacksquare$ $() \to “Q10Viking”$ This lambda has no parameters and returns a String as an expression

$\blacksquare$ $() \to { return \ “Q10Viking”; }$ This lambda has no parameters and returns a String( Using an explicit return statement)

注意两种返回字符串的方式

Functional interface

We can pass lambda expression as argument to a method,but the method must accept functional interface as parameter.

A functional interface is an interface that specifies exactly one abstract method

In the Java API such as Comparator and Runnable

What can you do with functional interfaces?

Lambda expressions let you provide the implementaion of the abstract method of a functional interface directly inline and treat the whole expression as an instance of a functional interface

function descriptor

The signature of the abstract method of the functional interface essentially describes the signature of the lambda expression.We call this abstract method a function descriptor.

  • The notation $() \to void$ represent a function with an empty list of parameters and returning $void$.(This is exactly what the $Runnable$ interface represents.)
  • $(Apple,Apple) \to int$ denotes a function taking two $Apple$s as parameters and returning an $int$.

@FunctionalInterface annotation

This annotation is used to indicate that the interface is intended to be a functional interface.

The function is the same as @Override notation to indicate that a method is overriden.
