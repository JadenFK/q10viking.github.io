Beautiful库详解

发表于 2018-09-12 | 分类于 python scraping

Beautifulsoup

  • 灵活又方便的网页解析库，处理高效，支持多种解析器
  • 利用它不用编写正则表达式即可方便地实现网页信息的提取

解析库

解析器 使用方法 优势 劣势
Python标准库 BeautifulSoup(markup, “html.parser”) Python的内置标准库、执行速度适中 、文档容错能力强 Python 2.7.3 or 3.2.2)前的版本中文容错能力差
lxml HTML 解析器 BeautifulSoup(markup, “lxml”) 速度快、文档容错能力强 需要安装C语言库
lxml XML 解析器 BeautifulSoup(markup, “xml”) 速度快、唯一支持XML的解析器 需要安装C语言库
html5lib BeautifulSoup(markup, “html5lib”) 最好的容错性、以浏览器的方式解析文档、生成HTML5格式的文档 速度慢、不依赖外部扩展

基本使用

标签选择器

$\blacksquare$ 选择元素

  • 直接制定标签名字

$\blacksquare$ 获取名称

  • .name指定标签的名称

$\blacksquare$ 获取属性

  • [‘attrs-name’],或则.attrs[“attrs-name”]

$\blacksquare$ 获取内容

  • .string 获取标签的内容

$\blacksquare$ 嵌套选择

  • 用.来层层分隔

$\blacksquare$ 子节点和孙节点

  • .contents 以列表的形式获取标签的子节点,包含其内容
  • .children 也是获取子节点，但是返回的是迭代器，我们用enumerate来进行遍历

$\blacksquare$ .descendants 获取所有子孙节点

$\blacksquare$ 父节点和祖先节点

  • 获取父节点 .parent

  • 获取所有的祖先节点 .parents

$\blacksquare$ 兄弟节点

  • .next_siblings 位于标签的后面的兄弟节点
  • .previous_siblings 位于标签前面的兄弟节点

标准选择器

  • 可根据标签名，属性，内容查找文档

$\blacksquare$ name参数

  • 根据标签名选择标签，以列表形式返回

  • 根据已经筛选出来的标签列表，再进行层层筛选

$\blacksquare$ attrs参数

$\blacksquare$ text参数

  • 根据文本内容进行选择，返回的文本内容而不是标签的列表
  • find返回单个元素，find_all返回所有元素

  • find_parents() find_parent()

    • find_parents()返回所有祖先节点，find_parent()返回直接父节点。

  • find_next_siblings() find_next_sibling()

    • find_next_siblings()返回后面所有兄弟节点，find_next_sibling()返回后面第一个兄弟节点。

  • find_previous_siblings() find_previous_sibling()

    • find_previous_siblings()返回前面所有兄弟节点，find_previous_sibling()返回前面第一个兄弟节点。

  • find_all_next() find_next()

    • find_all_next()返回节点后所有符合条件的节点, find_next()返回第一个符合条件的节点

  • find_all_previous() 和 find_previous()

    • find_all_previous()返回节点前所有符合条件的节点, find_previous()返回第一个符合条件的节点

CSS选择器

  • 通过select()直接传入CSS选择器即可完成选择，返回的标签列表
  1. class选择器在选择的时候前面加上.
  2. id选择器在其前面加上#
  3. 标签名称则直接写标签名
  4. 空格分隔进行层层迭代选择
  • [Example-1 基本使用]
  • [Exampe-2: for循环进行层层迭代选择]

$\blacksquare$ 获取属性

  • 通过attrs[]，或者直接[]

$\blacksquare$ 获取内容

  • 通过get_text()

总结

  • 推荐使用lxml解析，必要时使用html.parser
  • 标签选择器筛选功能弱但是速度快
  • 建议使用find(),find_all()查询匹配单个结果或者多个结果
  • 如果对CSS选择器熟悉建议使用select()
  • 记住常用的获取属性和文本值得方法
