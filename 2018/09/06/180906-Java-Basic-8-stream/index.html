Getting started with streams

Stream,a short definition is “a sequence of elements from a source that supports data processing operations”

  • $Sequence \ of \ elements$—Like a collection,a stream provides an interface a sequenced set of values of a specific element type.Because collections are data structures,they’re mostly about storing and accessing element with specific time/space complexities.However streams are about expressing computations.That’s Collections are about data;Streams are about computation.
  • $Source$—Streams consume from a data-providing source such as collections,arrays,or I/O resources.
  • $Data \ processing \ operations$—Streams support database-like operations and common operations from functional programming language to manipulate data.

In addtion,stream operations have two important characteristics:

  • $Pipeline$—Many stream operations return a stream themselves,allowing operations to be chained and form a larger pipeline.A pipeline of operations can be viewed as a database-like query on the data source.
  • $Internal \ iteration$—In contrast to collections,which are iterated explicitly using an iterator,stream operations do the iteration behind the scenes for you.

We start our discussion of streams with collections,because that’s the simplest way to begin working with streams.
