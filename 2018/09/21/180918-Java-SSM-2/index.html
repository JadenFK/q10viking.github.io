简介

  • MyBatis是基于Java的持久层框架
  • 内部封装了JDBC，不需要花费精力去处理诸如注册驱动，创建Connection,配置Statement等繁杂过程
  • 通过xml或者注解的方式将要执行的各种statement(statement,preparedStatement)配置起来
  • 由MyBatis框架执行SQL并将结果映射成java对象并返回

MyBatis与Hibernate

  • Hibernate框架是提供了全面的数据库封装机制的“全自动”ORM，即实现了POJO和数据库表之间的映射，以及SQL的自动生成和执行
  • MyBatis是“半自动化”的ORM,即是在POJO类与SQL语句之间的映射关系。（并不会自动生成SQL语句）
  • MyBatis具有以下特点
  1. 在XML文件中配置SQL，实现SQL语句与代码的分离，给程序的维护带来很大的便利
  2. 因为需要自己写SQL语句，可以根据数据库自身的特点灵活控制SQL语句。
  3. 因此能够实现比Hibernate等全自动化ORM框架更高的查询效率，能够完成复杂的查询

$\blacksquare$ XML配置文件

  • 分别在项目的org.q10viking.dao包下建立一个（可以是任意名称）mapper.xml文件，在src下建立（可以是任意名称）mybatis.xml文件
  • 与mybatis.xml文件与mapper.xml文件分别相关的约束文件，在解压后的mybatis.jar包下的org.apache.ibatis.builder.xml中有mybatis-3-mapper.dtd，mybatis-3-config.dtd
  • 根据文档分别在mybatis.xml和mapper.xml中配置对应的约束文件

约束文件dtd,xsd规定了该类型的xml文件中可以包含的标签，以及标签的书写顺序等。
