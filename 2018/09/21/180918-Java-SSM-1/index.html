JDBC原生编程的问题

  • 数据库连接，使用时就创建，不使用就立即释放，对数据库频繁的开启和关闭，造成数据库资源浪费，影响数据库性能。可以使用数据库连接池解决
  • 将sql语句硬编码到java代码中，如果sql语句修改，需要重新编译java代码，不利于系统维护。将sql语句配置在xml配置文件中，即使sql语句变化，也不需要对java代码进行重新编译
  • 将preparedStatement中设置参数，对占位符位置和设置参数值，硬编码在java代码中，不利于系统维护。
  • 从resultSet中遍历结果收集数据时，存在硬编码，将获取表的字段进行硬编码，不利于系统维护。可以将查询的结果集，自动映射成java对象
