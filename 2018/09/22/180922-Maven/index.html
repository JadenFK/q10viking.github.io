Apache Maven

  • A project management tool with a project object model（POM）.
  • A set of standards.
  • A project life cycle.
  • A dependency management system.
  • Logic for executing plugin goals at lifecycle phases.
  • Projects follow a consistent structure.
  • Projects are IDE agnostic
  • Maven allows for easy modifications to the project.
  • Maven simplifies the declaration of project dependencies.
  • It uses a project object model(POM) file.

Project Object Model(POM)

The POM file is the heart of Maven projects

POM

  • Source code must be in the src/main folder
  • Resources necessary for the project are in another folder.
  • Test cases are in a specifically name folder.
  • Target folder is used for the final JAR file.

A project module includes

  • Project description
  • Unique set of “coordinates”
  • Project attributes
  • Project’s license
  • Project version
  • Program authors and contributors
  • Dependencies

Such as

POM Categiries

  • The POM contains all of the information about a project.
  • The file is stored with an XML extension.
  • It has a groupId,artifactId and a version.Those threee things make up the Maven coordinates and are required for all projects.

Project relationships

  • Maven makes it easy to track down dependencies.
  • Example external relationships are Log4j and junit
  • Internal is where project-a depends on project-b
  • They are established using the Maven coordinates.
  • Relationships are described as dependencies.
  • Projects can inherit project relationships.

Maven lifecycle

Validate

  • Validate the project is correct.

Compile

  • Compile the source code of the project

Test

  • Test the compile source code using a unit testing framework.

Package

  • Package the compiled code

Integration-test

  • Deploy the package into an environment where intergration tests can be run.

Verify

  • Run any checks to verify the package is valid.

Install

  • Install the package into the local repository

Deploy

  • Copies the final package to the remote repository.

Maven Repository

  1. Central repository that contains open-source components.
    The Central Repository
  2. Local repository: Maven creates a local repository ~/.m2
  • When you download a dependency from a remote Maven repository,Maven stores a copy of the dependency in your local repository.
  • In addtion,it also places a copy of your jar file and the pom.xml file for each installed project.

Add dependencies

  • Maven supports both internal and external dependencies
  • Maven first checks the local repository
  • If not found,it downloads the dependency.
