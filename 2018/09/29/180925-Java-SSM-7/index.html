查询缓存

发表于 2018-09-29 | 分类于 SSM学习

一级缓存

  • MyBatis一级查询缓存是基于org.apache.ibatis.cache.impl.PerpetualCache类的HashMap本地缓存，其作用域是SqlSession.
  • 在同一个SqlSession中两次执行相同的Sql查询语句，第一次执行完毕后，会将查询结果写入到缓存中，第二次会从缓存中直接获取数据，而不再到数据库中进行查询。
  • 当SqlSession结束后，该SqlSession中的一级缓存也就不存在了。
  • MyBatis默认一级缓存是开启状态，且不能关闭

  • MyBatis的查询依据是：Sql的id+SQL语句

  • 从DB中进行select查询的条件是
    • 缓存中根本就不存在这个Key
    • 缓存中存在该key所对应的Entry对象，但是其value为null
二级缓存的配置

< cache />标签中的相关属性设置

  • eviction: 逐出策略。当二级缓存中的对象达到最大值时，就需要通过缓存策略将缓存中的对象移除缓存。

    • FIFO: First In First Out,先进先出
    • LRU: Least Recently Used,未被使用时间最长的

  • flushInterval: 刷新缓存的时间间隔，单位毫秒

  • readOnly: 设置缓存中数据是否只读。只读的缓存会给所有调用者返回缓存对象的相同实例，因此这些对象不能被修改，这提供了很重要的性能优势。但读写的缓存会返回缓存对象的拷贝。这会慢一些，但是安全，因此默认是false.

  • size: 二级缓存中可以存放的最多对象个数。默认为1024

二级缓存的关闭

$\blacksquare$ 全局关闭

  • 即整个应用的二级缓存全部关闭，所有查询均不使用二级缓存。
  • 在主配置文件如mybatis.xml的全局设置< settings / >中，该属性为cacheEnabled,设置为false,则关闭，设置为true,则开启。默认是开启的。

$\blacksquare$ 局部关闭

  • 局部关闭是指整个应用的二级缓存是开启的，但是针对某个< select / >查询，不使用二级缓存
  • 在要关闭二级缓存的< select / >标签中，将其属性useCache设置为false。
二级缓存的使用规则

  • 1) 只能在一个命名空间下使用二级缓存
    由于二级缓存的数据是基于namespace的，即不同namespace中的数据互不干扰。在多个namespace中若均对同一个表的操作，name这多个namespace中的数据就可能出现不一致现象。

  • 2) 在单表上使用二级缓存
    如果一个表与其他表有关联关系，那么就非常有可能存在多个namespace对同一数据的操作。而不同namespace中的数据互不干扰，所以就可能出现数据不一致现象

  • 3) 查询多于修改时使用二级缓存
    在查询操作远远多于增删改操作的情况下可以使用二级缓存。（任何增删改操作都将刷新二级缓存，对二级缓存的频繁刷新将会降低系统性能）

ehcache二级缓存

mybatis的特长是SQL操作，缓存数据管理不是其特长。为了提高缓存性能，myBatis允许使用第三方缓存产品，如ecache

启动ehcache缓存机制
  • 下载地址
  • 导入两个jar包，一个为ehcache的核心包，一个是myBatis与ehcache整合的插件Jar包
  • 解压ehcache的核心jar包，ehcache-core-2.6.8.jar，将其中一个配置文件ehcache-failsafe.xml直接放到项目的src目录下，并更名为ehcache.xml

  • 在映射文件的mapper中的< cache />中通过type指定缓存机制为ehcache缓存。默认为org.apache.ibatis.cache.impl.PerpetualCache

    该类在mybatis-encache的jar包中可以找到

ehcache.xml文件

  • $\blacksquare$ < diskStore />标签

    • 指定一个文件目录，当内存空间不够，需要将二级缓存中的数据写到硬盘上

    • java.io.tmpdir,表示当前系统的默认文件临时目录

    • 当前系统的默认文件临时目录，可以通过System.property()方法查看

  • $\blacksquare$ < defaultCache />标签

  • maxElementsInMemory: 指定该内存缓存区可以缓存对象的最多个数

  • eternal: 设定缓存对象是否会过期。若设为true,表示对象永远不会过期，此时会忽略timeToIdleSeconds与timeToLiveSeconds属性。
  • timeToIdleSeconds: 设定允许对象处于空闲状态的最长时间，以秒为单位。（当对象自从最近一次被访问后，若处于空闲状态的时间超过了设定的值，则为过期）。若设置为0,则对象可以无期限地处于空闲状态
  • timeToLiveSeconds: 设定对象存在缓存中的最长时间。注意只有timeToLiveSeconds $\ge$ timeToIdleSeconds,才有意义。
  • overflowToDisk: 设定为true,表示缓存对象到达了maxElementsInMemory界限，会将溢出的对象写到< diskStore >元素指定的硬盘目录中。
  • maxElementsOnDisk: 指定硬盘缓存区可以存放对象的最多个数
  • diskPersistent: 指当程序结束时，硬盘缓冲区的缓存对象是否做持久化。
  • diskExpiryThreadIntervalSeconds: 指定硬盘中缓存对象的失效时间间隔
  • memoryStoreEvictionPolicy: 如果内存缓存区超过限制，选择移向硬盘缓存区中的对象使用的策略。
    • FIFO: First In First Out
    • LFU: Less Frequently Used,最少使用
    • LRU: Least Recently Used,最近最少使用
ehcache在不同mapper中的个性化设置

ehcache.xml中设置的属性值，会对该项目中所使用ehcache缓存机制的缓存区域起作用。一个项目中可以有多个mapper,不同的mapper有不同的缓存区域起作用。一个项目可以有多个mapper,不同的mapper有不同的缓存区域，对于不同缓存区域也可以进行当前区域的个性化设置
