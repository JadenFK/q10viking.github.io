一对多关联查询

  • 在查询一方对象的时候，同时将其所关联的多方对象叶都查询出来

    下面以国家Country与部长Minister间的一对多关系进行演示

  • 为了便于演示，关注的一方对应多方，所以我们关注的是country

  • 注意： 在定义实体时，若定义的是双向关联，即双方的属性中均有对方对象作为域属性出现，那么他们在定义各自的toString()方法时需要注意，只一方可以输出另一方即可，不要让双方的toString（）方法均输出对方，这样会形成递归调用，导致程序出错。

  • Mytest.java
[方法二：] 多表单独查询方式
  • 多表连接查询是将多张表进行连接，连为一张表后进行查询。其查询的本质是一张表。
  • 而多表单独查询方式是将多张表各自查询各自的相关内容，需要多张表联合数据，即将主表的查询结果联合其他表的查询结果，封装为一个对象。

  • mapper.xml

  • Mytest.java

多对一关联查询

  • 在查询对方对象的时候，同时将其所关联的一方对象也查询出来。
  • 由于查询对方对象时也是一个一个查询，所以多对一关联查询，其实也就是一对一关联查询。

自关联查询

也就是递归查询

  • 自关联是指，自己既充当一方，又充当多方，是1：n或n:1的变型。
  • 对于新闻栏目NewsColumn,可以充当一方，即父栏目，也可以充当多方，即子栏目。
  • 反映到DB表中，只有一张表，这张表中具有一个外键，用于表示该栏目的父栏目。
  • 一级栏目没有父栏目，所以可以将其外键值设为0，而子栏目则具有外键值
NewsLabel表

由于Column是DBMS中的关键字，为了避免误解，将新闻栏目实体类定义为NewsLabel.

多对多查询

  • 多对多关系由两个互反的两个一对多关系构成
  • 有外键的是多方，反过来，多方必有外键
  • 一个学生可以选择多门课程，而一门课程可以有多个学生选择。这就是典型的多对多关系。
  • 多对多关系都会通过一个中间表来建立，如选课表
定义实体
  • 在定义双向关联的实体的toString()方法时，只让一方的toString()方法可以输出对方，不要让双方均可输出对方。

  • 否则将会出现输出时的递归现象，导致报错
