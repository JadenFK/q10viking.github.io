注册中央处理器
  • 全限定性类名

该中央调度器是一个servlet，名称为DispatcherServlet.中央调度器的全限定性类名在导入spring-webmvc-5.0.9.RELEASE.Jar文件

  • < load-on-startup/ >

在< servlet / >中添加< load-on-startup / >的作用是，标记是否在Web服务器（这里是Tomcat）启动时会创建这个Servlet实例。即是否在Web服务器启动时调用执行该Servlet的init（）方法，而不是在真正访问的时候创建。

  • < url-pattern / >

对于< url-pattern / >，不能写为/ ,最好也不要写为/，建议是.do

  • 配置文件位置与名称

系统默认加载配置sprinmvc文件的位置是在WEB-INF目录下，找名称为Servlet名称-servlet.xml的配置文件。这里的“Servlet名称”是指的是注册中央调度器时< servlet-name / >标签中指定的Serlvet的name值。

而一般情况下，该配置文件是我们任意命名放置在src目录下的饿。所以，在注册中央调度器时，我们需要设置查找SpringMVC配置文件的位置
