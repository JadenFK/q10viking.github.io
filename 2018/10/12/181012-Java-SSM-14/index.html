顾问Advisor

发表于 2018-10-12 | 分类于 SSM学习
  • GIthub-相关代码
  • 通知（Advice）是Spring提供的一种切面（Aspect）,但其功能过于简单，只能将切面织入到目标类的所有目标方法中，无法完成将切面织入到指定目标方法中去。
  • 顾问(Advisor)是Spring提供的另一种切面。其能够完成更为复杂的切面织入功能。
  • PointcutAdvisor是顾问的一种，可以指定具体的切入点。

名称匹配方法切入点顾问

  • org.q10viking.aop01
  • NameMatchMethodPointcutAdvisor，名称匹配方法切入点顾问。容器可根据配置文件中指定的方法名来设置切入点。
  • 匹配的是简单方法名

只对两个方法进行织入

  • applicationContext.xml

  • MyTest.java

正则表达式方法切入点顾问

  • org.q10viking.aop02
  • RegexpMethodPointcutAdvisor,容器根据正则表达式来设置切入点

  • 匹配的是全限定性方法名 如： org.q10viking.aop02.UserServiceImp.doFirst

  • applicationContext.xml

  • MyTest.java

问题

当前代码存在的两个问题

  1. 若存在多个目标对象，就需要使用多次ProxyFactoryBean来创建多个代理对象，这会使得配置文件变得臃肿
  2. 用户真正想调用的是目标对象，而真正可以调用的却是代理对象，这不符合正常逻辑。

以上这两个问题，均为ProxyFactoryBean类的功能太简单引起。
