AspectJ对AOP的实现

发表于 2018-10-12 | 分类于 SSM学习
  • Github-相关代码
  • 很多框架都实现了AOP这种编程思想，除了Spring，还有AspectJ框架
  • 因为AspectJ也实现了AOP功能，实现方式更为简捷，并且支持注解式开发
  • Spring已经将AspectJ的AOP的实现引入到了自己的框架中。

在Spring中使用AOP开发时，一般使用AspectJ的实现方式

  • 目标对象，切入点表达式
  • 切面-advice或advisor

AspectJ的切入点表达式

  • 代表就是目标对象，只不过涉及到了更具体，到了方法层面
  • 切入点表达式要匹配的对象就是目标的方法名
  • 表达式中加[ ]的部分表示可以省略
  • 各个部分用空格分开
  • 返回类型与方法名是必须的
  • 可以使用以下符号
例子

  • 指定只有一级包下的service子包下所有类中的doSome()方法为切入点

  • 指定所有包下的service子包下所有类中的doSome()方法为切入点

AspectJ开发环境

  • 根据文档添加AOP约束文件，AOP schema
  • 导入spring-aspects-jar包
  • 以及com.springsource.org.aspectj.weaver-1.6.8.RELEASE.jar

前置通知@Before

@Before前置通知-增强方法中有JoinPoint参数

  • 在目标方法执行之前执行，被注解为前置通知的方法，可以包含一个JoinPoin类型参数
  • 该类型的对象本身就是切入点表达式。
  • 通过该参数可以获得，切入点表达式，方法签名，目标对象等

后置通知@AfterReturning

@AfterReturning 后置通知-注解有 returning 属性

  • 目标方法执行之后，可以获取目标方法的返回值。
  • 该注解的returning属性就是用于指定接收方法返回值的变量名的。
  • 所以被注解为后置通知的方法，除了可以包含JoinPoint参数外，还可以包含用于接收返回值的变量
  • 该变量最好定义为Object类型，因为目标方法可能返回任何类型

@Around 环绕通知-增强方法有 ProceedingJoinPoint 参数

  • 包含一个ProceedingJoinPoint类型的参数
  • 接口ProceedingJoinPoint其有一个proceed()方法，用于执行目标方法，若有返回值，该方法返回值就是目标方法的返回值。
  • 被注解为环绕增强的方法的返回类型为Object

@AfterThrowing 异常通知-注解中有 throwing 属性

  • 目标方法抛出异常后，该注解的throwing属性用于指定所发生的异常类对象
  • 被注解为异常通知的方法可以包含一个参数Throwable,参数名为throwing指定的名称，表示发生异常的对象

@After最终通知

  • 无论方法是否出现异常，该通知都会执行

@Pointcut 定义切入点

  • 当较多的通知增强方法使用相同的execution切入点表达式时，编写，维护均较为麻烦。
  • AspectJ提供了@Pointcut注解，用于定义execution切入点表达式

用法

  • @Pointcut注解在一个方法之上（该方法一般使用private的标识方法，即没有实际作用的方法）
  • 以后所有的execution的value属性值均可使用该方法名作为切入点。
