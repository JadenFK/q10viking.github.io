Spring与Web

发表于 2018-10-16 | 分类于 SSM学习
  • Github-相关项目
  • 在web项目中使用Spring框架，关键是要解决在web层（指Servlet）中获取到Spring容器。

Spring的Web插件

Spring容器的创建问题

  • 在Web应用中，ServletContext对象是唯一的，一个应用，只有一个ServletContext对象。该对象是在Web应用装载时初始化的。
  • 若将spring容器的创建时机，放在ServletContext初始化时，就可以保证Spring容器的创建只会执行一次，也就保证了Spring容器在整个应用中唯一性。

*Spring容器获取问题

  • Spring容器创建好以后，在整个Web应用的生命周期中，Spring容器应该是随时可以被访问的。即Spring容器应该具有全局性
  • 而ServletContext对象的域属性，就具有应用的全局性。
  • 所以将创建好的Spring容器以域属性的形式放入到ServletCOntext的域空间中，就保证了Spring容器的全局性。

上述工作，已经被spring-web.jar包实现

如何使用详解

导入spring-web-5.0.9.RELEASE.jar
注册监听器ContextLoaderListener
  • 要在ServletContext初始化化时创建Spring容器，就需要使用监听器接口ServletContextListener对ServletContext进行监听。

  • spring-web.jar包实现了该org.springframework.web.context.ContextLoaderListener类，

  • 在web.xml中注册该监听器

  • ContextLoaderListener类完成了两个重要的工作： 1）创建容器对象。 2）将容器对象放到ServletContext的域属性空间中。

  • 查看ContextLoaderListener的源码，一共有四个方法，两个是构造方法，一个初始化方法，一个销毁方法。

  • 在这四个方法中较为重要的方法应该是contextInitialized()

    跟踪initWebApplicationContext（）方法，可以看到创建了容器对象

    并且将创建好的容器对象放入到ServletContext的域属性空间中，key为一个常量：WebApplicationContext.ROOT_WEB_APPLICATION_CONTEXT_ATTRIBUTE

指定Spring配置文件的位置< context-param / >
  • ContextLoaderListener在对Spring容器进行创建时，需要加载Spring配置文件。其默认的Spring配置文件位置为名称： WEB-INF/applicationContext.xml。

  • 但是我们的项目，一般将该配置文件放在classpath下，即src.所以需要在web.xml中对spring配置文件的位置及名称进行指定

    根据initWebApplicationContext上的注释

    以及要读取的配置文件位置参数名称

获取Spring容器

直接从ServletContext中获取
  • 从源码分析中知道ServletContext的域属性存放着key为WebApplicationContext.ROOT_WEB_APPLICATION_CONTEXT_ATTRIBUTE

  • 可以直接通过ServletContext的getAttribute（）方法获取容器对象

  • 使用WebApplicationContext代替ApplicationContext

通过WebApplicationContextUtils获取
  • key 太难记
  • 工具类WebApplicationContextUtils有一个方法专门用于从ServletContext中获取Spring容器对象，
  • getRequiredWebApplicationContext(ServletContext sc)
