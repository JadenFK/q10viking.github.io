DHCP配置

发表于 2018-05-03 | 分类于 Internet

在Windows server 2003进行DHCP的配置

搭建目标

DHCP工作流程

四线会话

  1. DHCP Discover: 客户端在网络中搜索服务器。
    源地址：0.0.0.0；目的地址：255.255.255.255通过广播，携带者本机的MAC地址

  2. DHCP Offer: 服务器向客户端响应服务
    还是以广播的形式

  3. DHCP Request: 客户机端向目标服务器端发送服务请求
    还是广播的形式：通知其他主机这个IP我即将占用

  4. DHCP Ack：服务器相客户端提供服务
    携带者租约信息，保留着该IP信息

安装DHCP

首先提供DHCP服务的主机必须有固定的IP,必须是要基于server的系统，如windows server 2003.

配置作用域

将要分配给客户端的IP地址

激活作用域

使得客户端能够使用

步骤

将之前在电脑上的DHCP删除掉

start->Manage Your Server->Add and remove a role

设置自己的静态IP地址

添加DHCP Component

start->Control Panel->Add or Remove Programs->Add/Remove Windows Components


在弹出的窗口中选择Networking Services->DHCP,单击OK
完成安装。

安装完成之后在start->Administrative Tools中可以看到DHCP选项，点击它。

添加域

右键服务器名称选择New Scope

为这个域添加名称和描述

设置地址范围：

设置在地址范围中不能分给客户端的IP地址


设置默认过期时间为8天

最后一步选择No

激活域

问题

  1. 虚拟机的网络配置，需要为内部网络，高级选择：混杂模式：全部允许，以形成一个局域网LAN

  2. 在添加DHCP组件的时候，需要提前将IP设置为静态IP

windows-server-2003操作

手机驱动

当插入手机数据线时，windows会自动安装与手机相应的驱动 m3 note。

有一次插上以后，却显示无法识别USB设备，于是通过控制面板的电源选项–>更改计算机睡眠时间–>更改高级设置–>USB设置–>USB选择性暂停设置，将已启动改为已禁用，然后保存，接着再将其改为已启动，就可以了。系统就会重新安装

当在设备管理器中看到：Android Device-Android Composite ADB Interface时就可以了
