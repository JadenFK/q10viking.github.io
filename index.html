注入分类

  • Bean实例调用无参构造器创建了空值对象后，就要对Bean对象的属性进行初始化。
  • 初始化是由容器自动完成的，称为注入
  • 根据注入方式的不同，常用的有： 设置注入和构造注入
  • 还有另外一种，实现特定接口注入，由于这种方式采用侵入式编程，污染了代码，所以几乎不用。
设值注入
  • 设置注入是指，通过setter方法传入被调用者的实例。
  • 普通属性-value
  • 域属性-ref

命名空间注入

  • 对于设值注入与构造注入在配置文件中，除了使用 < property / >或 < constructor-arg />标签外，还可以使用命名空间注入方式，让注入的值以< bean />标签属性的方式出现。

  • 根据注入实现方式的不同： 分为p命名空间注入与c命名空间注入

    • p命名空间注入： 采用设置注入方式，故需要相应的setter
    • c命名空间注入： 采用构造注入方式，故需要相应的构造器
p命名空间设置注入
  • 按照官方文档添加 xmlns:p=”http://www.springframework.org/schema/p"
    • 通过检索p-namespace

  • 在< bean / >标签中使用p命名空间中的属性

    • p:bean 属性=”值” 说明： 该值为普通值
    • p:bean 属性-ref=”值” 说明： 该值为其它bean id.

  • applicationContext.xml

c命名空间构造注入
  • 添加 xmlns:c=”http://www.springframework.org/schema/c"
    • 通过在官网搜索 c-namespace

  • 在< bean />标签中使用c命名空间

    • c:bean参数=”值” 说明：该值为普通值
    • c:bean参数-ref=”值” 说明：该值为其他bean id

  • applicationContext.xml

prototype pattern(原型模式)

发表于 2018-10-07 | 分类于 Java基础

Prototype Design Pattern

  • The Prototype design pattern is used to specify the kinds of objects to create using a prototypical instance,and create new objects by copying this prototype.
  • The concept is to copy an existing object rather than creating a new instance from scratch,something that may include costly operations.
  • The existing object acts as a prototype and contains the state of the object.The newly copied object may change same properties only if required.
  • This approach saves costly resources and time,especially when the object creation is a heavy process.

  • In Java,there are certain ways to copy an object in order to create a new one.One way to achieve this is using the Cloneable interface.
  • Java provides the clone method,which an object inherits from the object class.
  • You need to implement the Cloneable interface and override this clone method according to your needs.

封装

装

  • 便于对数据的传递，在装在一个对象中

封

  • 设置外界对数据访问的权限

继承

都是从内存空间的解读来看

  • 对成员变量的继承，是对内存空间的复制，数据的拷贝
  • 对方法的继承，是使用权限的继承，而不是代码片段的继承（空间没有增加）
  • 子类的方法覆盖是在子类的方法区中又开辟了一个新的空间，并不是对父类方法区中的方法覆盖
    • 注意static方法： 在多态中，static覆盖是不起作用的，总会调用父类声明的静态方法。

  • Cat.java

  • Main.java

  • 编译时的静态绑定与运行时的动态绑定。

    • 继承
    • 重写

查询缓存

发表于 2018-09-29 | 分类于 SSM学习
  • 一级缓存的作用域是SqlSession
  • 二级缓存的作用域是整个应用
  • Github-相关代码

一级缓存

  • MyBatis一级查询缓存是基于org.apache.ibatis.cache.impl.PerpetualCache类的HashMap本地缓存，其作用域是SqlSession.
  • 在同一个SqlSession中两次执行相同的Sql查询语句，第一次执行完毕后，会将查询结果写入到缓存中，第二次会从缓存中直接获取数据，而不再到数据库中进行查询。
  • 当SqlSession结束后，该SqlSession中的一级缓存也就不存在了。
  • MyBatis默认一级缓存是开启状态，且不能关闭
